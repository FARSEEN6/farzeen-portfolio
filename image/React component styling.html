In React, you can style components in several ways from inline CSS to modern modular approaches. Heres a complete overview

1. Inline Styles

Apply styles directly to elements using a JavaScript object.

jsx
function Button() {
  const style = {
    backgroundColor: "blue",
    color: "white",
    padding: "10px 20px",
    border: "none",
    borderRadius: "8px"
  };

  return <button style={style}>Click Me</button>;
}

Pros:

  • Simple for small components.

  • Dynamically change styles with JS.

Cons:

  • No hover or pseudo-class support (:hover, :focus, etc.).

2. CSS Stylesheet

Create a .css file and import it into your component.

Button.css

css
.button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 8px;
}

Button.jsx

jsx
import "./Button.css";

export default function Button() {
  return <button className="button">Click Me</button>;
}

Pros:

  • Supports all CSS features (hover, media queries).

  • Easy to use.

Cons:

  • Class name conflicts possible (use CSS Modules for that).

3. CSS Modules

Scoped CSS for individual components.

Button.module.css

css
.button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 8px;
}

Button.jsx

jsx
import styles from "./Button.module.css";

export default function Button() {
  return <button className={styles.button}>Click Me</button>;
}

Pros:

  • CSS is locally scoped (no conflicts).

  • Supports full CSS syntax.

Cons:

  • Slightly more setup if mixing with global styles.

4. Styled Components (CSS-in-JS)

Use a library like styled-components.

bash
npm install styled-components

Button.jsx

jsx
import styled from "styled-components";

const Button = styled.button`
  background-color: blue;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 8px;

  &:hover {
    background-color: darkblue;
  }
`;

export default function App() {
  return <Button>Click Me</Button>;
}

Pros:

  • Scoped automatically.

  • Conditional and dynamic styling.

  • Full CSS power.

Cons:

  • Adds a dependency.

  • Slightly slower in very large apps.

5. Tailwind CSS (Utility-First)

Add Tailwind CSS for fast styling using classes.

bash
npm install -D tailwindcss
npx tailwindcss init

Button.jsx

jsx
export default function Button() {
  return (
    <button className="bg-blue-500 text-white px-4 py-2 rounded-lg hover:bg-blue-600">
      Click Me
    </button>
  );
}

Pros:

  • Very fast UI building.

  • Responsive & hover utilities built in.

Cons:

  • Learning curve for class names.

  • Inline class clutter in JSX.

6. Dynamic / Conditional Styling

You can also apply styles conditionally:

jsx
function Button({ primary }) {
  const className = primary ? "btn btn-primary" : "btn";
  return <button className={className}>Click Me</button>;
}

Or with inline:

jsx
style={{ backgroundColor: primary ? "blue" : "gray" }}
